# 사용자 스토리 (요구사항 도출)

1. 문서 목적
  본 문서는 사용자 관점에서 필요한 기능과 요구사항을 도출하기 위해 작성하였다. 도출된 사용자 스토리는 프로젝트의 요구사항 분석, 설계 및 구현 과정의 기초 자료로 활용한다. 또한 사용자 요구사항의 우선순위를 정의하여 Sprint 계획 수립에 활용한다.

2. 사용자(Persona)
사용자: 일반 사용자
설명:
- 자신의 수입과 지출을 기록하고 관리하고 싶은 사용자
- 카테고리별 예산을 설정하여 계획적인 소비를 관리하고 싶은 사용자
- 월별 예산 사용 현황과 남은 예산을 확인하고 싶은 사용자

3. 사용자 스토리

| ID    | 기능           | 우선순위 | 사용자 스토리                                                                        |
| ----- | -------------- | -------- | ------------------------------------------------------------------------------------ |
| US-01 | 회원가입       | High     | 사용자는 회원가입을 할 수 있다.                                                      |
| US-02 | 로그인         | High     | 사용자는 로그인을 할 수 있다.                                                        |
| US-03 | 전체 예산 설정 | High     | 사용자는 월별 예산을 설정할 수 있다.                                                 |
| US-04 | 카테고리 관리  | High     | 사용자는 카테고리 생성, 조회, 수정, 삭제를 할 수 있다.                               |
| US-05 | 수입 관리      | High     | 사용자는 수입 내역을 등록하고 관리할 수 있다.                                        |
| US-06 | 지출 관리      | High     | 사용자는 지출 내역을 등록하고 관리할 수 있다.                                        |
| US-07 | 남은 예산 조회 | High     | 사용자는 카테고리별 남은 예산, 전체 남은 예산, 실제 남은 잔액을 함께 확인할 수 있다. |
| US-08 | 월별 내역 조회 | Medium   | 사용자는 월별 수입/지출 내역과 예산 사용 현황을 조회할 수 있다.                      |
