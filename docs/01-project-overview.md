# 프로젝트 개요 (프로젝트 소개)

## 1. 프로젝트 소개

기존 가계부 애플리케이션은 소비 내역을 기록하는 기능에 초점을 맞추고 있어, 예산을 먼저 설정하고 계획적으로 소비를 관리하기에는 아쉬움이 있었다. 이러한 불편함을 해결하기 위해 카테고리별 예산을 설정하고 남은 예산을 직관적으로 확인할 수 있는 예산 중심 가계부를 기획하였다.

본 프로젝트는 Spring Boot와 REST API를 기반으로 개발하는 개인 프로젝트이며, 단순히 기능을 구현하는 것에 그치지 않고 요구사항 분석, 설계, 구현, 테스트, 회고까지 소프트웨어 공학 프로세스를 적용하여 진행한다. 이를 통해 실무와 유사한 개발 경험을 쌓고 취업 포트폴리오로 활용하는 것을 목표로 한다.

## 2. 프로젝트 차별점

### 2.1 예산 중심 관리
- 기존 가계부는 소비 내역 기록을 중심으로 관리하는 경우가 많다.
- 본 프로젝트는 카테고리별 예산을 먼저 설정한 뒤 소비를 기록하고, 남은 예산을 관리하는 방식을 제공한다.

### 2.2 사용자 맞춤형 카테고리
- 사용자가 자신의 소비 패턴에 맞게 카테고리를 직접 생성/수정/삭제할 수 있다.
- 획일적인 카테고리가 아닌 개인에게 맞는 예산 관리 환경을 제공한다.

### 2.3 예산과 실제 잔액 동시 제공
- 카테고리별 남은 예산과 전체 남은 예산을 자동으로 계산하여 제공한다.
- 예산 기준 남은 금액과 실제 남은 잔액을 함께 제공한다.
- 사용자는 계획한 소비와 실제 자금 현황을 동시에 확인할 수 있다.

## 5. 주요 기능

### 5.1 회원 관리
- 회원가입 및 로그인을 통해 사용자별 데이터를 관리한다.

### 5.2 예산 관리
- 월별 예산을 설정한다.
- 카테고리별 예산을 설정하고 관리한다.
- 전체 및 카테고리별 남은 예산을 자동으로 계산한다.

### 5.3 카테고리 관리
- 사용자가 원하는 카테고리를 생성, 수정, 삭제할 수 있다.

### 5.4 수입/지출 관리
- 수입과 지출 내역을 입력하고 관리한다.
- 입력된 내역을 카테고리와 연결하여 관리한다.

### 5.5 월별 조회
- 월별 수입 및 지출 내역을 조회한다.
- 해당 월의 예산 사용 현황을 확인한다.

### 5.6 REST API
- Next.js Frontend와 Spring Boot Backend 간 통신을 위해 REST API를 구현한다.

## 6. 기대 효과

### 6.1 사용자 관점
- 카테고리별 예산을 기반으로 계획적인 소비 습관을 형성할 수 있다.
- 남은 예산을 직관적으로 확인하여 효율적인 예산 관리가 가능하다.
- 사용자 맞춤형 카테고리를 통해 개인의 소비 패턴에 맞는 가계부를 사용할 수 있다.

### 6.2 개발자 관점
- Spring Boot와 REST API를 활용한 웹 애플리케이션 개발 경험을 쌓을 수 있다.
- 요구사항 분석부터 설계, 구현, 테스트, 회고까지 소프트웨어 공학 프로세스를 적용한 프로젝트를 수행할 수 있다.
- GitHub를 활용한 형상 관리 및 프로젝트 관리 경험을 쌓을 수 있다.
- 취업 포트폴리오로 활용 가능한 개인 프로젝트를 완성할 수 있다.
